# Gromacs 核心计算热点分析报告

## 1. Gromacs概述

GROMACS (GROMOS Molecular Dynamics) 是一款广泛使用的分子动力学模拟软件，主要用于生物大分子体系（蛋白质、核酸、脂质等）的模拟计算，是HPC领域最重要的应用之一。

## 2. Gromacs核心计算阶段

典型的分子动力学模拟每一时间步包含以下主要计算阶段：

```
邻居列表构建 
    ↓
非键相互作用计算 (力场计算)
    ↓
键相互作用计算
    ↓
PME傅里叶变换 (长程静电)
    ↓
约束算法 (LINCS/SHAKE)
    ↓
速度和位置更新
    ↓
数据输出
```

## 3. 核心计算热点分析

### 3.1 力场计算 (Non-bonded Interactions)

**计算占比**：通常占总计算时间的 **60%-80%**，是最大的热点。

#### 计算内容：
- 范德华相互作用计算（Lennard-Jones势）
- 短程静电相互作用计算
- 对每个原子对计算能量和力

#### 计算特点：
- 计算密集型，每个时间步需要计算大量原子对
- 局部性好，基于邻居列表只计算截止距离内的相互作用
- 高度并行化，每个原子/块可以独立计算
- 包含大量简单算术运算（乘加、指数计算等）

#### 计算pattern：
```
for each atom i:
  for each neighbor j of i within cutoff:
    r = distance between i and j
    compute van der Waals energy/force
    compute short-range electrostatic energy/force
    accumulate to i and j
```

### 3.2 傅里叶变换 (PME - Particle Mesh Ewald)

**计算占比**：通常占总计算时间的 **10%-30%**，取决于体系大小和并行设置。

#### 计算内容：
- 将电荷分配到三维网格
- 3D FFT计算倒空间
- 在倒空间计算长程静电相互作用
- 逆FFT变换回实空间
- 插值得到原子受力

#### 计算特点：
- 核心是3D FFT操作，计算复杂度O(N log N)
- 大规模并行时，FFT需要全局通信，成为可扩展性瓶颈
- 网格计算部分具有良好的数据局部性
- 包含大量规整的张量操作（网格插值、卷积）

#### 计算步骤：
```
1. 电荷散布(Spreading): 将原子电荷插值到网格 ← 可融合
2. 3D FFT: 实空间 → 倒空间
3. 倒空间势能计算: 与格林函数卷积 ← 可融合
4. 逆FFT: 倒空间 → 实空间
5. 力插值: 从网格插值到原子 ← 可融合
```

### 3.3 约束算法 (LINCS/SHAKE)

**计算占比**：通常占总计算时间的 **5%-15%**。

#### 计算内容：
- 约束键长保持（如氢键的固定长度）
- LINCS (Linear Constraint Solver) 是Gromacs默认算法
- 通过迭代方式约束原子位置

#### 计算特点：
- 稀疏矩阵乘法（LINCS投影操作）
- 迭代收敛过程，步数通常较少（2-4步）
- 每个约束链计算相对独立
- 大量的向量-矩阵乘法操作

### 3.4 邻居列表构建

**计算占比**：通常占总计算时间的 **2%-5%**（每n步更新一次）。

#### 计算内容：
- 根据原子坐标构建邻居列表
- 使用Verlet列表或网格方法
- 只保留截止距离内的原子对

#### 计算特点：
- 网格排序和桶排序
- 距离计算比较
- 数据重排操作

## 4. 各热点计算占比参考

下表给出典型生物大分子体系（百万原子级）的计算时间分布：

| 计算组件 | 单核/单GPU | 多节点并行 | 计算模式 |
|----------|------------|------------|---------|
| 非键相互作用 | 70-80% | 50-65% | 计算密集 |
| PME (含FFT) | 10-20% | 20-30% | 计算+通信 |
| 约束算法 | 5-10% | 5-10% | 稀疏计算 |
| 邻居列表 | 2-5% | 2-4% | 数据整理 |
| 其他（键合、更新） | 3-5% | 3-6% | 控制流 |

> 数据来源：Gromacs性能分析文献和超算最佳实践

## 5. 当前Gromacs在GPU/AI加速器上的实现特点

### 5.1 现有优化

- **非键相互作用**：已经充分GPU加速，使用SIMT/SIMD并行
- **PME**：整体搬移到GPU，减少CPU-GPU数据传输
- **约束算法**：GPU并行实现
- **邻居列表**：Verlet截断，GPU上动态更新

### 5.2 存在的优化空间

1. **多个小kernel启动开销**：当前阶段划分导致多个独立kernel启动
2. **中间结果数据移动**：阶段之间需要写回全局内存
3. **PME中多个小操作未能充分融合**：电荷散布、势能计算、力插值等步骤存在融合机会
4. **非对称计算负载的负载不均衡**：不同Tile之间计算量差异较大

## 6. 可识别的计算Pattern识别

### 6.1 适合PTO优化的Pattern

| 计算阶段 | Pattern类型 | 可融合性 |
|----------|------------|---------|
| 非键相互作用 | 邻域对计算 + 归约 | ⭐⭐⭐⭐⭐ |
| PME电荷散布 | 散射-聚集操作 | ⭐⭐⭐⭐ |
| PME势能计算 | 网格点张量运算 | ⭐⭐⭐⭐⭐ |
| PME力插值 | 聚集-散射操作 | ⭐⭐⭐⭐ |
| LINCS约束 | 稀疏矩阵向量乘 + 迭代 | ⭐⭐⭐ |
| 邻居列表构建 | 排序 + 距离过滤 | ⭐⭐ |

### 6.2 Pattern特征分析

**计算密集型Pattern（非键相互作用）**：
- 每个Tile包含固定数量原子对
- 计算与访存比高
- 适合在PTO中作为基本计算单元

**网格插值Pattern（PME）**：
- 规则三维网格访问
- 每个网格点计算独立
- 非常适合Tile划分

**稀疏线性代数Pattern（LINCS）**：
- 不规则内存访问
- 计算密度较低
- 仍可在Tile层面进行粗粒度融合

## 7. 结论

Gromacs的四个主要计算热点中：

1. **非键相互作用**是最大热点，计算占比最高，具有规则的并行Pattern
2. **PME**是第二大热点，包含多个可融合的子步骤
3. **约束算法和邻居列表**占比较小，但仍存在优化空间
4. 大部分计算都可以表示为张量/Tile操作，适合PTO范式进行融合优化
