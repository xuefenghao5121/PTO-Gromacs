### 关键发现
1. **最佳线程数: 8**
   - 这对应 i7-13700 的 P-core 数量
   - 超过 8 线程后，E-core 参与导致性能下降
   
2. **小系统扩展性差**
   - 216 水分子系统太小
   - 通信开销随线程数增加而主导
   
3. **混合架构挑战**
   - Intel 13 代混合架构 (P-core + E-core)
   - E-core 参与反而降低整体性能

## 5. 优化建议

### 针对当前硬件
1. **使用 8 线程运行**
   ```bash
   export OMP_NUM_THREADS=8
   gmx mdrun -ntomp 8 ...
   ```

2. **绑定到 P-core**
   ```bash
   # 仅使用 P-core (核心 0-7)
   taskset -c 0-7 gmx mdrun ...
   ```

3. **编译优化版本**
   - 当前 APT 版本仅使用 SSE4.1
   - 从源码编译可启用 AVX2
   - 预期额外 20-30% 性能提升

### 针对更大系统
- 系统规模 > 10,000 原子时，扩展性会显著改善
- 建议使用 GPU 加速 (当前系统无 GPU)
